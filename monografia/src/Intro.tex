\chapter{Introdução}
Cada vez mais a sociedade moderna está dependente dos computadores para realização de suas atividades do cotidiano. Com isso a internet se tornou um item essencial em qualquer lugar.

São compras realizadas e contas pagas sem a necessidade de sair de casa. Tudo que é necessário é um computador e uma conexão com a internet. Esses serviços que facilitam tanto a vida vão acumulando informações sobre os usuários, como localização, dados financeiros, e outros dados pessoais ou informações confidenciais e os usuários acreditam que essas informações estão bem protegidas. Mas isso nem sempre é verdade.

Tome como exemplo a \emph{PSN} (\estrangeiro{PlayStation Netowork}), a rede utilizada pelo PlayStation 3 para organizar partidas \estrangeiro{online}, para comprar e baixar jogos, comunicação entre os usuários, dentre outros recursos. Recentemente uma vulnerabilidade foi encontrada nessa rede e vários dados de usuários foram obtidos por quem a explorou. Entre esses dados estavam informações sobre os cartões de crédito utilizados para fazer compras na PSN.

O caso da PSN mostra que mesmo os maiores serviços existentes estão sujeitos a todo tipo de vulnerabilidade e nem sempre ferramentas tradicionais como \estrangeiro{firewalls} e criptografia são o suficiente para protege-las.

Os \estrangeiro{scanners} de vulnerabilidade são ferramentas que fazem a análise das aplicações \estrangeiro{web} de forma automática. São capazes de detectar e reportar vulnerabilidades de segurança\cite{Tania:2010:Scanners}.

O objetivo deste trabalho é analisar a eficiência de alguns desses \estrangeiro{scanners} de vulnerabilidade para encontrar um tipo de vulnerabilidade específico: \estrangeiro{XPath Injection}.
