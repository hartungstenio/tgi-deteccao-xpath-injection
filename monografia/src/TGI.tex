\begin{resumo}
Cada vez mais as empresas buscam disponibilizar seus produtos pela internet, pelo alcance possível e pela facilidade de entrega e implantação. Já existe uma grande variedade de produtos disponibilizados na \estrangeiro{web}. Vemos pacotes de produtividade para escritório, como é o caso do \estrangeiro{Google Docs} (Google) e do \estrangeiro{Office 360} (Microsoft), sistemas de gestão empresarial que funcionam pelo navegador de internet, armazenamento de dados em servidores remotos, etc. Com a popularização da internet e o advento de várias tecnologias, abriu-se uma ampla área para expansão no fornecimento de software e serviços.

Porém, uma das maiores preocupações das empresas fornecedoras e, principalmente, dos usuários desses serviços é a segurança das informações que estão armazenadas. Se existir alguma vulnerabilidades de segurança, um usuário mal-intencionado pode encontrar uma forma de obter todos esses dados para uso pessoal. \estrangeiro{Scanners} de vulnerabilidade são ferramentas capazes de identificar essas vulnerabilidades. O objetivo deste trabalho é avaliar a segurança de aplicações web em relação à vulnerabilidades do tipo \emph{XPath Injection} e também avaliar a eficácia de \estrangeiro{scanners} de vulnerabilidades na detecção dessas vulnerabilidades.

\begin{description}
	\item[Palavras-chave:] segurança, aplicação \estrangeiro{web}, serviço \estrangeiro{web}, \estrangeiro{scanner} de vulnerabilidade
\end{description}
\end{resumo}

\begin{abstract}
More and more companies want to make their products available over the internet for so they can reach more costumers and the easiness to delivery and deploy. There are many products available on the web. We see office productivity packages such as Google Docs (by Google) and the Office 360 (by Microsoft), enterprise resource planning systems that run in the internet browser, data storage on remote servers, etc.

However, one of the biggest concerns of suppliers and, especially, users of these services is the security of information that are stored. If there are any security vulnerabilities, a malicious hacker can find a way to get all this data for personal use. Web Application vulnerability scanners are tools capable of identifying those vulnerabilities. This work tries to evaluate the security of web application with respect of the XPath Injection vulnerability and also assess the effectiveness of vulnerability scanners to detect those vulnerabilities.

\begin{description}
	\item[Keywords:] security, web application, web service, vulnerability scanner
\end{description}
\end{abstract}
